# GVIM Documentation

GVIM is an AI Agent Platform designed specifically for Chemistry and Materials Science. It provides a runtime harness for composing scientific agents with advanced capabilities—including RDKit molecular analytics, pymatgen structural calculation, Materials Project database retrieval, Ketcher chemistry canvas, and safe sandbox execution. It also provides an application layer that turns these scientific capabilities into a usable and collaborative web product experience.

This documentation is organized around those two parts:

- **GVIM Harness**: the core SDK and runtime layer for building your own scientific agent system.
- **GVIM App**: a reference application built on top of the Harness for science workflows, chemistry analytics, and researcher collaboration.

If you want to understand how GVIM works, start with the Introduction. If you want to build on the core runtime, go to the Harness docs. If you want to deploy and use GVIM as an application, go to the App docs.

### Introduction

The Introduction section helps you build the right mental model before you look at implementation details.

- **What is GVIM** explains what GVIM is and what problems it is designed to solve.
- **Why GVIM** explains the motivation behind the project.
- **Core Concepts** introduces the concepts that appear across the documentation.
- **Harness vs App** explains the relationship between the runtime layer and the application layer.

### GVIM Harness

The Harness section is the core of the technical documentation. It is written for developers who want to build their own GVIM-based system.

- **Quick Start** shows how to create your first harness with the core GVIM API.
- **Design Principles** explains the architectural ideas behind the Harness.
- **Configuration** covers the main configuration surface of the SDK/runtime.
- **Memory**, **Tools**, **Skills**, and **Sandbox** explain the major system capabilities separately.
- **Customization** shows how to adapt GVIM to your own product requirements.
